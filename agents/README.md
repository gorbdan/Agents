# Агент-бот для Сырника

Telegram-бот с тремя AI-агентами. Загружает код Сырника с GitHub и анализирует через Claude.

## Команды

| Команда | Что делает |
|---------|-----------|
| `/qa` | Тестирует все сценарии пользователей |
| `/qa <тема>` | Тест конкретного флоу |
| `/analyze` | Ищет баги и проблемы в коде |
| `/analyze <тема>` | Анализ конкретной части |
| `/fix` | Патчи для топ-5 проблем |
| `/fix <проблема>` | Фикс конкретной проблемы |
| `/reload` | Перезагрузить код с GitHub |

## Деплой на BotHost

### 1. Создай нового бота
Напиши @BotFather → /newbot → получи токен.

### 2. Получи Anthropic API ключ
Зайди на platform.anthropic.com → API Keys → Create Key.

### 3. Залей репозиторий на GitHub
Положи эти файлы в новый репозиторий (агент-бот — отдельный репо от Сырника).

### 4. Подключи репо в BotHost
Так же как Сырник — "Update from Git".

### 5. Добавь переменные окружения в BotHost

Обязательные:
```
AGENT_BOT_TOKEN=токен_от_botfather
ANTHROPIC_API_KEY=sk-ant-...
GITHUB_REPO=gorbdan/sirnike
ADMIN_IDS=320423776
```

Опциональные (если репо приватное):
```
GITHUB_TOKEN=ghp_...
```

### 6. Запусти

BotHost сам соберёт Docker-образ и запустит `python agent_bot.py`.

## Как работает

1. При старте бот загружает SirNike.py, config.py, db.py с GitHub
2. Код хранится в памяти как контекст для агентов
3. Каждая команда отправляет код + задачу в Claude Sonnet
4. Агент анализирует и отвечает прямо в Telegram

Если обновил код Сырника → напиши `/reload` → агенты увидят изменения.

## Примеры

```
/qa онбординг нового пользователя
/analyze race conditions в очереди генерации
/analyze логика возврата изюминок при ошибке
/fix потеря изюминок при краше воркера
/fix реферальный бонус начисляется повторно
```
